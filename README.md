# ⚖️ Court Transcriber — транскрипция заседаний с разделением спикеров

Full-stack приложение для расшифровки длинных аудиозаписей (судебные заседания,
совещания) с **диаризацией** — кто и когда говорил. Локальный ASR на GPU,
определение смены говорящих, юридический словарь для терминов, веб-интерфейс,
очередь задач и экспорт готового протокола.

## Что решает

Ручная расшифровка многочасового заседания — это дни работы секретаря.
Приложение делает черновой протокол автоматически: грузишь запись →
получаешь текст, разбитый по спикерам и репликам, с возможностью править и экспортировать.

## Архитектура

```
client/ (web UI)  ──►  server/ (FastAPI)  ──►  очередь задач
                                │
                                ├─► audio_preprocess (нормализация, тишина)
                                ├─► WhisperX (large-v3, GPU) ─► текст + тайминги
                                ├─► диаризация: NeMo Sortformer / pyannote ─► спикеры
                                ├─► legal_vocabulary ─► коррекция юр-терминов
                                └─► export ─► готовый протокол
```

Свой бэкенд, своя очередь, сменные ASR- и диаризационные движки, доменный
словарь и веб-клиент — то есть полный цикл, а не обёртка над одним API.

## Технические решения

- **Сменные бэкенды ASR** — `whisperx` (локально на GPU, бесплатно) / `yandex`
  (SpeechKit) / `mock` (тесты UI без GPU). Выбор через `.env`, код один.
- **Сменная диаризация** — NeMo **Sortformer** (language-agnostic, по эмбеддингам
  голоса, сильнее на длинных записях и overlap) с fallback на pyannote.
- **Очередь задач** — длинные записи обрабатываются асинхронно, UI не висит.
- **Доменный словарь** (`legal_vocabulary.py`) — пост-коррекция юридических
  терминов, которые общий Whisper путает.
- **Аутентификация и профили** — multi-user, разные профили обработки.

## Стек

`Python` · `FastAPI` · `WhisperX` · `NeMo Sortformer` · `pyannote` · `PyTorch` · `CUDA` · `JavaScript` (web-клиент) · очередь задач

## Структура

```
server/   FastAPI-бэкенд: main, auth, queue, ASR, диаризация, export, словарь
client/   веб-клиент: загрузка, просмотр по спикерам, правка, экспорт
scripts/  настройка окружения
```

## Результат

- Полный конвейер «аудио → протокол по спикерам» локально на GPU
- Поддержка многочасовых записей через асинхронную очередь
- 3 сменных ASR-бэкенда и 2 движка диаризации под разные сценарии
